# Módulo 11 — Seguridad y Gobernanza (Pareto 20/80)

Objetivo general:
Aplicar prácticas esenciales de seguridad y gobernanza en pipelines de datos: control de acceso, gestión de secretos, RLS, enmascarado de PII y auditoría mínima.

Resultados de aprendizaje:
- Configurar roles y permisos en el warehouse / Databricks / Snowflake.
- Implementar Row-Level Security (RLS) en SQL y Power BI / consulta.
- Gestionar secretos de forma segura (Vault, Databricks secrets, GitHub Secrets).
- Documentar lineage y políticas básicas de governance.

Duración estimada: 6–10 horas

Pareto 20/80:
- 20%: RLS patterns, secrets usage, least privilege, basic auditing.
- 80%: aplicar RLS en un dataset, configurar secrets y policy, mapear lineage básico.
